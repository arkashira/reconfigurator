# Business Model Canvas – reconfigurator

| **Section** | **Details** |
|-------------|-------------|
| **Value Proposition** | • **Instant, AI‑driven reconfiguration** of physical systems (e.g., HVAC, robotics, smart‑home appliances) without manual CAD or firmware updates.<br>• **Zero‑downtime deployment**: the tool generates configuration scripts and firmware patches that can be streamed to devices over the air.<br>• **Cost savings**: reduces engineering hours by 60 % and eliminates the need for on‑site visits.<br>• **Scalable integration**: works with any device that exposes a standard API or CAN/Modbus interface, making it suitable for industrial, automotive, and consumer IoT. |
| **Customer Segments** | 1. **Industrial Automation** – factories, warehouses, and logistics centers looking to re‑optimize equipment layout.<br>2. **Smart‑Home OEMs** – manufacturers of HVAC, lighting, and security systems that need rapid firmware updates.<br>3. **Robotics Startups** – companies building modular robots that require on‑the‑fly reconfiguration.<br>4. **Facilities Management** – large campuses and data centers that need to re‑route HVAC or power distribution without downtime. |
| **Channels** | • **Direct sales** to large OEMs and industrial accounts via the Axentx sales team.<br>• **Marketplace integration**: listing on the Axentx “Product Hub” and on third‑party IoT platform marketplaces (e.g., AWS IoT Marketplace, Azure Marketplace).<br>• **Developer portal**: open‑source SDK and API docs hosted on GitHub (arkashira/reconfigurator) with community support.<br>• **Channel partners**: system integrators and IoT solution providers that embed reconfigurator into their offerings. |
| **Revenue Streams** | • **Subscription (SaaS)**: monthly/annual plans based on number of devices and configuration complexity.<br>• **Per‑device licensing**: one‑time fee for perpetual use in embedded deployments.<br>• **Professional services**: consulting, custom integration, and training (hourly rates).<br>• **Marketplace revenue share**: 20 % cut on any third‑party plugins or extensions built on top of reconfigurator. |
| **Cost Structure** | • **Engineering & R&D**: core ML model training, firmware generation pipeline, and API development.<br>• **Cloud infrastructure**: GPU compute for inference, storage for device metadata, and CI/CD pipelines.<br>• **Support & Ops**: 24/7 monitoring, customer success, and SLA enforcement.<br>• **Sales & Marketing**: account executives, partner enablement, and marketing collateral.<br>• **Compliance & Security**: audit, penetration testing, and certification costs. |
| **Key Resources** | • **ML models**: trained on the 7.3 M auto + 6.4 M instr‑resp pairs, fine‑tuned for device‑specific configuration generation.<br>• **Open‑source SDK**: reusable libraries for device discovery, OTA updates, and safety checks.<br>• **Knowledge base**: curated dataset of device APIs, firmware schemas, and safety constraints.<br>• **Engineering team**: ML engineers, firmware developers, and DevOps specialists.<br>• **Partnership network**: OEMs, system integrators, and cloud providers. |
| **Key Activities** | • **Model training & fine‑tuning** on device‑specific data.<br>• **API & SDK development** for seamless integration with physical components.<br>• **Continuous integration of new device types** and safety validation.<br>• **Customer onboarding & support** (tutorials, documentation, live demos).<br>• **Marketplace and partner ecosystem management**. |
| **Key Partners** | • **Device OEMs** (e.g., HVAC, robotics, automotive) that provide APIs and firmware access.<br>• **Cloud providers** (AWS, Azure, GCP) for scalable inference and OTA delivery.<br>• **System integrators** that embed reconfigurator into larger automation stacks.<br>• **Compliance bodies** (UL, IEC) for safety certification.<br>• **Academic partners** for cutting‑edge research on safe ML‑driven configuration. |
